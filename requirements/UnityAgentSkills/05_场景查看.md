# 需求文档 - 场景查看功能

## 1. 项目现状与核心目标

### 1.1 用户需求简述

需要提供 3 个命令,用于场景打开与场景内对象查询:

1. `scene.open`: 打开指定场景(支持 Single/Additive).
2. `scene.queryHierarchy`: 查询当前已加载场景中的 GameObject 层级.
3. `scene.queryComponents`: 查询指定场景内某个 GameObject 的组件详情.

### 1.2 一致性原则

本需求要求与 `prefab.queryHierarchy` 和 `prefab.queryComponents` 保持一致的使用习惯:

- 参数命名尽量复用现有 prefab 风格.
- `nameContains` 的过滤行为与 `matches[]` 字段保持一致.
- 组件查询的过滤参数与返回字段保持一致.

### 1.3 核心目标

- 外部工具可通过 UnityAgentSkills 批量命令协议完成场景查看.
- 在 Play 模式和编辑模式都可执行“查看”类命令.
- 仅在编辑模式允许执行场景打开,避免自动退出 Play 模式.
- 在多场景加载(Additive)情况下,查询结果可明确区分所属场景.

## 2. 范围与边界

### 2.1 功能点简述

#### 命令 1: scene.open

- [ ] 接收 `scenePath`(必须以 `Assets/` 开头).
- [ ] 支持可选参数 `openMode`,取值 `Single` 或 `Additive`,默认 `Single`.
- [ ] 打开成功时返回轻量结果: `scenePath`, `sceneName`.
- [ ] 如果当前是 Play 模式,必须直接报错,不自动退出 Play 模式.

#### 命令 2: scene.queryHierarchy

- [ ] 查询范围为"当前已加载的所有场景".
- [ ] Play 模式下,查询范围还须包含 `DontDestroyOnLoad` 场景(Unity 隐藏的不销毁场景,标准 `SceneManager` API 无法枚举,需特殊处理).
- [ ] 默认返回树结构.
- [ ] 支持 `nameContains` 模糊过滤.
- [ ] 传入 `nameContains` 时不返回树,改为返回扁平 `matches[]`.
- [ ] `matches[]` 每项字段必须与 prefab 侧 1:1 一致:`name`,`instanceID`,`path`,`siblingIndex`,`depth`,`isActive`.
- [ ] 为区分多场景,结果按场景分组,外层返回 `scenes:[{sceneName,...}]`.
- [ ] 即使只加载 1 个场景,也必须返回 `sceneName`.

#### 命令 3: scene.queryComponents

- [ ] 查询目标定位方式与 prefab 一致: `objectPath + siblingIndex`.
- [ ] 必须新增必填参数 `sceneName`,用于指定查询场景.
- [ ] `sceneName` 支持传入 `"DontDestroyOnLoad"` 以查询不销毁场景中的对象(Play 模式下).
- [ ] 支持组件类型模糊过滤(`componentFilter`,包含匹配,大小写不敏感),默认返回所有组件(包括内置组件).
- [ ] 返回字段风格与 prefab.queryComponents 一致,并包含 `sceneName`.

### 2.2 模式约束

- 查看命令(`scene.queryHierarchy`,`scene.queryComponents`)在 Play 模式和编辑模式都允许执行.
- 打开命令(`scene.open`)仅允许在编辑模式执行.
- Play 模式调用 `scene.open` 时,返回错误:
  - `error.code = ONLY_ALLOWED_IN_EDIT_MODE`.
  - `error.message` 需对非专业人士可读,例如"当前为 Play 模式,仅编辑模式可打开场景".
  - `error.detail` 必须包含 `OnlyAllowedInEditMode` 关键字,便于调用方做机器识别.
- 为保证该错误码可稳定落地,实现侧需要将 `ONLY_ALLOWED_IN_EDIT_MODE` 纳入框架统一错误码常量与异常映射链路.

### 2.3 排除项

- 不包含任何场景编辑能力(如新增/删除对象,改组件参数,保存场景).
- 不支持通过 `instanceID` 直接查询组件.
- 不支持正则匹配,`nameContains` 仅支持 contains 语义.
- 不在本需求中定义“切换 activeScene”的额外命令.

## 3. 输入输出要求(非代码,协议说明)

### 3.1 输入协议

批量协议整体格式沿用 `01_整体与框架需求.md`,本节仅描述各命令 `params`.

#### scene.open params

- `scenePath`: string,必填,`Assets/` 开头.
- `openMode`: string,可选,`Single|Additive`,默认 `Single`.

#### scene.queryHierarchy params

- `includeInactive`: bool,可选,默认 `true`.
- `maxDepth`: int,可选,默认 `-1`(无限深度),合法值范围 `>= -1`.
- `nameContains`: string,可选.
- `maxMatches`: int,可选,默认 `50`,仅在 `nameContains` 生效时使用,合法值范围 `> 0`.

说明:

- 本命令不接收 `scenePath`.
- 本命令不接收 `sceneName` 过滤参数.
- `nameContains` 匹配规则与 prefab 保持一致: 对输入先做 Trim,空串视为未传,按 IgnoreCase 执行 contains 匹配.

#### scene.queryComponents params

- `sceneName`: string,必填.
- `objectPath`: string,必填.
- `siblingIndex`: int,可选,默认 `0`.
- `componentFilter`: string[],可选,默认返回所有组件(包括内置组件).
- `includePrivateFields`: bool,可选,默认 `false`.

**componentFilter 匹配规则**:

- 支持**模糊匹配**(包含匹配,大小写不敏感)
- **OR 关系**: 只要组件名称包含任意一个 filter 字符串就匹配
  - 示例: `["transform", "ui"]` 会匹配 `Transform`, `RectTransform`, `UIButton`, `UILabel` 等
- **null 或空数组**: 返回所有组件(包括 `Transform`, `RectTransform` 等内置组件)
- **空字符串**: 匹配所有组件(等同于不传该参数)

说明:

- `sceneName` 匹配规则为大小写敏感,与 Unity `Scene.name` 属性对应.
- `objectPath` 表示 `sceneName` 场景内的 GameObject 层级路径,使用 `/` 分隔,首段必须是该场景的 root GameObject 名称.
- 当同一路径下存在多个同名对象时,使用 `siblingIndex` 做唯一定位,规则与 prefab.queryComponents 一致.
- 错误码风格: 参数缺失或无效时返回 `error.code = INVALID_FIELDS`,并在 `error.message` 中明确字段名,例如 `"sceneName is required"`.

### 3.2 输出结构要求

#### scene.open result

- `scenePath`: string
- `sceneName`: string

#### scene.queryHierarchy result

- 默认模式(未传 `nameContains`):
  - `loadedSceneCount`: int
  - `scenes`: array
    - 每项包含 `sceneName`,`scenePath`(可选),`totalGameObjects`,`hierarchy`
    - `scenePath` 用于规避同名场景歧义,当场景仅通过 `sceneName` 无法区分时提供.
    - `hierarchy` 为树节点数组,每个节点字段固定为 `name`,`instanceID`,`path`,`siblingIndex`,`depth`,`isActive`,`children`.
    - `children` 字段必须存在且类型恒为 array,无子节点时返回 `[]`,禁止 null 或缺省.
- 过滤模式(传入 `nameContains`):
  - `loadedSceneCount`: int
  - `scenes`: array
    - 每项包含 `sceneName`,`totalMatches`,`matchedCount`,`truncated`,`matches`

过滤模式字段语义:

- `totalMatches`: 未截断前的总命中数.
- `matchedCount`: 本次实际返回的 `matches` 条数,即 `matches.length`,且 `<= maxMatches`.
- `truncated`: 当 `totalMatches > matchedCount` 时为 `true`,否则为 `false`.

`matches[]` 每项字段固定为:

- `name`,`instanceID`,`path`,`siblingIndex`,`depth`,`isActive`

#### scene.queryComponents result

- `sceneName`: string
- `objectPath`: string
- `instanceID`: int
- `totalComponents`: int
- `components`: array
  - 每项字段与 prefab 保持一致:`type`,`instanceID`,`scriptPath(可选)`,`properties`

## 4. 举例覆盖需求和边缘情况

### 例 1: 编辑模式打开场景(默认 Single)

输入示例:

```json
{
  "type": "scene.open",
  "params": {
    "scenePath": "Assets/Scenes/Main.unity"
  }
}
```

预期:

- 成功打开 Main 场景.
- 返回 `scenePath` 和 `sceneName`.

### 例 2: 编辑模式 Additive 打开场景

输入示例:

```json
{
  "type": "scene.open",
  "params": {
    "scenePath": "Assets/Scenes/Battle.unity",
    "openMode": "Additive"
  }
}
```

预期:

- 新场景以 Additive 方式加载.
- 后续 `scene.queryHierarchy` 可看到多个场景分组.

### 例 3: Play 模式下调用 scene.open

输入示例:

```json
{
  "type": "scene.open",
  "params": {
    "scenePath": "Assets/Scenes/Main.unity"
  }
}
```

预期:

- 命令失败.
- 返回 `error.code = ONLY_ALLOWED_IN_EDIT_MODE`.
- 不自动退出 Play 模式.

### 例 4: 查询层级(不带 nameContains)

输入示例:

```json
{
  "type": "scene.queryHierarchy",
  "params": {
    "includeInactive": true,
    "maxDepth": -1
  }
}
```

预期:

- 返回所有已加载场景.
- 每个场景一组 `sceneName + hierarchy`.
- 单场景时也保留 `sceneName`.

### 例 5: 查询层级(带 nameContains)

输入示例:

```json
{
  "type": "scene.queryHierarchy",
  "params": {
    "nameContains": "Button",
    "includeInactive": false,
    "maxMatches": 20
  }
}
```

预期:

- 不返回 `hierarchy`.
- 返回按场景分组的 `matches[]`.
- `matches[]` 每项字段严格为 `name`,`instanceID`,`path`,`siblingIndex`,`depth`,`isActive`.

### 例 6: 查询组件(标准)

输入示例:

```json
{
  "type": "scene.queryComponents",
  "params": {
    "sceneName": "Main",
    "objectPath": "Main/Panel_Content/UIButton_Confirm",
    "siblingIndex": 0,
    "componentFilter": ["UIButton", "transform"],
    "includePrivateFields": false
  }
}
```

预期:

- 在 `Main` 场景中定位对象.
- 返回所有名称包含 "UIButton" 或 "transform" 的组件(模糊匹配,大小写不敏感).
- 如 "transform" 会匹配到 `Transform`, `RectTransform` 等.
- 如 "UIButton" 会匹配到 `UIButton`, `UIButtonGroup` 等.

### 例 7: 漏传 sceneName

输入示例:

```json
{
  "type": "scene.queryComponents",
  "params": {
    "objectPath": "Main/Panel_Content/UIButton_Confirm"
  }
}
```

预期:

- 命令失败.
- 返回参数缺失错误,明确提示 `sceneName is required`.

### 例 8: 同路径对象依赖 siblingIndex 区分

场景层级中同一父节点下有两个同名对象:

- `UIButton_Confirm`(siblingIndex=0)
- `UIButton_Confirm`(siblingIndex=2)

输入示例:

```json
{
  "type": "scene.queryComponents",
  "params": {
    "sceneName": "Main",
    "objectPath": "Main/Panel_Content/UIButton_Confirm",
    "siblingIndex": 2
  }
}
```

预期:

- 只命中 siblingIndex=2 的对象.
- 定位规则与 prefab.queryComponents 一致.

## 5. 验收检查清单(面向非专业审查)

- [ ] `scene.open` 在编辑模式可用,Play 模式必报错且不退出 Play.
- [ ] `scene.open` 支持 `openMode=Single|Additive`.
- [ ] `scene.queryHierarchy` 不接收 `scenePath`,默认查所有已加载场景.
- [ ] `scene.queryHierarchy` 单场景和多场景都能明确看到 `sceneName`.
- [ ] `nameContains` 模式下返回 `matches[]`,且字段与 prefab 完全一致.
- [ ] `scene.queryComponents` 必填 `sceneName`,并使用 `objectPath + siblingIndex` 定位.
- [ ] 组件查询过滤参数与 prefab 一致,返回字段风格一致.
