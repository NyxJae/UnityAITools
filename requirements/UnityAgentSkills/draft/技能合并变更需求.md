# 临时变更需求 — 技能合并为 unity-agent-skills

> ⚠️ 本文档为临时指导文档，改造完毕后删除。不记录过程，只描述变更目标。

## 1. 变更概述

将 SkillsExporter 从「导出 9 个独立技能文件夹」改为「导出 1 个统一技能包 `unity-agent-skills`」。

## 2. 目标技能结构

```
unity-agent-skills/
├── SKILL.md                              # 主文档
│   ├── 脚本使用说明
│   ├── log.query / log.screenshot / log.refresh（完整参数+示例+原理+工作流）
│   └── 能力目录（名称 + 命令清单 + reference 路径指针）
├── scripts/
│   └── execute_unity_command.py          # 共用脚本（模板注入 Commands 路径）
└── references/
    ├── prefab-view.md
    ├── prefab-edit.md
    ├── scene-view.md
    ├── scene-edit.md
    ├── playmode.md
    ├── prefab-bridge.md
    ├── editor-action.md
    └── editor-control.md
```

各 reference 末尾包含指向相关 reference 的路径指针。

## 3. 代码变更

### 3.1 删除

- `SkillsExporter/SkillConfigs/` 下全部 `SkillConfig_*.cs`（9 个文件）
- `SkillConfigsRegistry.cs`（重构为轻量索引）

### 3.2 新增

- `SkillsExporter/SkillFiles/` 目录，布局与导出结构一致：
  ```
  SkillFiles/
  ├── SKILL.md
  ├── execute_unity_command.py
  └── references/
      ├── prefab-view.md
      ├── prefab-edit.md
      ├── scene-view.md
      ├── scene-edit.md
      ├── playmode.md
      ├── prefab-bridge.md
      ├── editor-action.md
      └── editor-control.md
  ```

- 轻量注册类（替代 `SkillConfigsRegistry`）：只存 `SkillFiles/` 根目录路径，导出时遍历复制整个目录。
- `execute_unity_command.py` 中保留 `{AGENT_COMMANDS_DATA_DIR}` 占位符，导出时做字符串替换后写入目标。

### 3.3 SkillsExporter UI 改动

- 移除 9 个技能复选框 + 全选/取消全选
- 简化为：导出路径显示 + 修改按钮 + 导出按钮
- 点击导出 → 遍历 `SkillFiles/` → 复制全部文件到 `<exportPath>/skills/unity-agent-skills/` → 对 `.py` 做路径注入

### 3.4 旧配置兼容

不保留旧 `EditorPrefs` 键的兼容，导出路径沿用现有 key 格式。
