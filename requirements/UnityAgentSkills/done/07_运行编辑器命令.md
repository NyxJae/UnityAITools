# 需求文档 - 运行编辑器命令

## 1. 项目现状与核心目标

### 1.1 用户需求简述

用户希望通过统一命令入口,触发 Unity Editor 内开发者本来就能触发的动作,包括:

- 菜单栏可触发动作.
- 工具窗口按钮可触发动作.
- 快捷键可触发动作.
- 用户新写的编辑器方法.

用户明确诉求:

- 不要求调用者去看代码命名空间细节.
- 希望系统自动扫描可调用方法.
- 命令侧仍沿用 `type=editor.runAction`.
- `actionId` 使用完整限定名 `Namespace.ClassName.MethodName`.

### 1.2 当前能力与可行性结论

结合现有实现,本需求可行:

- 已有命令执行链路: 命令注册 -> 按 `type` 分发 -> 结果回写.
- 已有插件加载链路: CoreCommands + 反射插件命令.
- 已有可复用动作入口,例如重启命令服务菜单对应方法.

结论:

- 本需求基于现有框架扩展 `editor.runAction` 的动作发现与映射规则.
- 不需要调用者手动查找代码并填写命名空间后再调用.

### 1.3 技术方案概要

- 主链路: `editor.runAction` -> `CommandHandlerRegistry` -> EditorAction 执行器.
- 动作来源: 自动扫描符合暴露条件的方法,生成动作目录.
- 动作标识: `actionId=Namespace.ClassName.MethodName`.
- 执行层优先使用 Unity Editor API 和现有方法调用.
- 反射仅用于“发现和建立映射”,不允许任意方法直接反射执行.

## 2. 范围与边界

### 2.1 功能范围(必须)

- [ ] 新增编辑器动作命令化能力,覆盖菜单,按钮,快捷键,插件公开方法.
- [ ] 新能力必须复用现有批次协议,输入仍为 `batchId + commands[]`.
- [ ] 每条命令必须遵循 `id + type + params` 结构.
- [ ] 命令类型固定为 `editor.runAction`.
- [ ] `actionId` 固定为完整限定名 `Namespace.ClassName.MethodName`.

### 2.2 自动扫描规则(必须)

- [ ] 扫描范围仅限项目约定的编辑器程序集范围.
- [ ] 仅将满足“可暴露条件”的方法加入动作目录.
- [ ] 扫描结果在命令服务启动和重启后可刷新.
- [ ] 若方法被移除,对应 `actionId` 在下一次刷新后不可再调用.

### 2.3 可暴露条件(首版)

- 仅允许 `public static` 方法.
- 方法所在类必须是编辑器工具域(非运行时业务逻辑类).
- 方法参数必须可从 `actionArgs` 安全映射.
- 方法标记为高风险或命中黑名单时,不得暴露或不得执行.

### 2.4 高风险排除项(首版)

**类别规则:**

- 删除类: 删除资产,批量删除对象,清空目录.
- 覆盖类: 强制覆盖导入,大范围覆盖配置.
- 破坏性运行态类: 无确认的强制停止关键运行流程.

**黑名单示例(完整限定名):**

- `UnityEditor.AssetDatabase.DeleteAsset`.
- `UnityEditor.FileUtil.DeleteFileOrDirectory`.
- `Project.Tools.EditorOps.ClearDirectory`.
- `Project.Tools.EditorOps.ForceOverwriteImport`.
- `Project.Tools.EditorOps.OverwriteGlobalConfig`.

命中黑名单时:

- 该命令返回 `status=error`.
- 错误码使用 `RUNTIME_ERROR`.
- 错误文案包含"命中高风险排除项".

### 2.5 排除项(明确不做)

- 不做运行时游戏逻辑调用(仅覆盖 Editor 动作).
- 不做绕过命令注册表的旁路执行.
- 不做调用者手工填写程序集或反射签名才能执行的模式.

## 3. 输入输出约束(必须与框架协议一致)

### 3.1 输入结构

本能力输入沿用批次协议:

- 顶层字段:
  - `batchId`: 批次唯一标识.
  - `commands`: 命令数组.
- 每条命令字段:
  - `id`: 命令唯一标识.
  - `type`: 固定为 `editor.runAction`.
  - `params`: 参数对象:
    - `actionId`: string,格式 `Namespace.ClassName.MethodName`.
    - `actionArgs`: object,可选,动作参数.

### 3.2 输出结构

- 批次级状态沿用: `processing/completed/error`.
- 命令级状态沿用: `success/error`.
- 命令失败时,错误码至少覆盖:
  - `UNKNOWN_TYPE`: `type` 未注册.
  - `INVALID_FIELDS`: 参数缺失或参数类型错误.
  - `RUNTIME_ERROR`: `actionId` 不存在,执行异常,或命中高风险排除项.

## 4. 举例覆盖需求和边缘情况

### 例 1: 调用已存在方法(重启命令服务)

pending 输入示例:

```json
{
  "batchId": "batch_editor_action_001",
  "commands": [
    {
      "id": "cmd_restart_service",
      "type": "editor.runAction",
      "params": {
        "actionId": "UnityAgentSkills.UI.SkillsExporterWindow.RestartCommandServiceMenu"
      }
    }
  ]
}
```

预期结果:

- 批次 `status=completed`,该命令结果为 `success`.
- 返回可读结果信息.

### 例 2: 用户新增方法后自动可调用

场景: 用户新增 `Project.Tools.RefreshActions.ReimportAllConfigs`.

pending 输入示例:

```json
{
  "batchId": "batch_editor_action_002",
  "commands": [
    {
      "id": "cmd_reimport_configs",
      "type": "editor.runAction",
      "params": {
        "actionId": "Project.Tools.RefreshActions.ReimportAllConfigs"
      }
    }
  ]
}
```

预期结果:

- 若方法满足可暴露条件且扫描已刷新,返回 `success`.
- 调用者不需要查命名空间以外的反射细节.

### 例 3: `actionId` 不存在

pending 输入示例:

```json
{
  "batchId": "batch_editor_action_003",
  "commands": [
    {
      "id": "cmd_unknown_action",
      "type": "editor.runAction",
      "params": {
        "actionId": "Project.Tools.NotExists.DoSomething"
      }
    }
  ]
}
```

预期结果:

- 返回 `status=error`.
- 错误码 `RUNTIME_ERROR`.
- 错误文案提示"actionId 不存在或不可调用".

### 例 4: 命中高风险黑名单

pending 输入示例:

```json
{
  "batchId": "batch_editor_action_004",
  "commands": [
    {
      "id": "cmd_blocked_delete",
      "type": "editor.runAction",
      "params": {
        "actionId": "UnityEditor.AssetDatabase.DeleteAsset",
        "actionArgs": {
          "path": "Assets/Test/A.prefab"
        }
      }
    }
  ]
}
```

预期结果:

- 直接拒绝执行.
- `status=error`,`error.code=RUNTIME_ERROR`.
- `error.message` 包含"命中高风险排除项".

### 例 5: 批次内部分成功

pending 输入示例:

```json
{
  "batchId": "batch_editor_action_005",
  "commands": [
    {
      "id": "cmd_ok",
      "type": "editor.runAction",
      "params": {
        "actionId": "Project.Tools.RefreshActions.ReimportAllConfigs"
      }
    },
    {
      "id": "cmd_fail",
      "type": "editor.runAction",
      "params": {
        "actionId": "UnityEditor.AssetDatabase.DeleteAsset",
        "actionArgs": {
          "path": "Assets/Test/B.prefab"
        }
      }
    }
  ]
}
```

预期结果:

- 第一条成功,第二条失败.
- 批次 `status=completed`.
- 统计 `totalCommands=2`,`successCount=1`,`failedCount=1`.

## 5. 验收口径

- [ ] 单命令和多命令都必须使用批次结构(`batchId + commands[]`).
- [ ] `actionId` 统一使用 `Namespace.ClassName.MethodName`.
- [ ] 新增合规编辑器方法后,扫描刷新即可被命令调用.
- [ ] `actionId` 不存在,参数错误,高风险命中三类失败稳定可复现.
- [ ] 批次内部分成功时,后续命令继续执行.
- [ ] 不与 `01_整体与框架需求.md` 与 `06_PlayMode_UI交互.md` 冲突.
