# 需求文档 - 预制体编辑功能 03(GameObject 创建与组件 CRUD)

## 1. 项目现状与核心目标

### 1.1 用户需求简述

需要在现有 `prefab.*` 命令族基础上,补齐以下 4 个命令,实现预制体内 GameObject 与组件的完整编辑闭环:

1. `prefab.createGameObject`: 在预制体中创建 GameObject.
2. `prefab.setComponentProperties`: 修改组件属性,支持简单类型,枚举,数组,嵌套对象,引用类型.
3. `prefab.addComponent`: 给目标 GameObject 添加组件.
4. `prefab.deleteComponent`: 删除目标 GameObject 上的组件.

### 1.2 一致性原则

本需求必须与 `03_预制体查看.md`,`04_预制体编辑01.md`,`04_预制体编辑02.md`,`05_场景查看.md` 保持一致的核心协议习惯:

- GameObject 定位统一使用 `objectPath + siblingIndex`.
- `prefabPath` 统一为 `Assets/` 开头的工程相对路径.
- 对子物体插入位置,统一使用 `insertSiblingIndex`/`targetSiblingIndex` 语义,避免与同名定位 `siblingIndex` 混淆.
- 错误码采用框架白名单机制,若新增错误码,必须同步扩展错误码常量与白名单映射.
- 本文新增错误码的强约束优先作用于本次新增的 4 个命令,存量命令按迁移策略逐步对齐.

### 1.3 核心目标

- 支持在预制体中从零创建 GameObject 层级.
- 支持对通用 Unity 组件和自定义组件进行属性级修改.
- 支持组件增删改查闭环,可与 `prefab.queryComponents` 联动使用.
- 支持引用类型的完整协议: `null`,`prefab 内对象`,`prefab 内组件`,`项目资源(assetPath)`.

## 2. 范围与边界

### 2.1 功能点简述

#### 命令 1: prefab.createGameObject

- [ ] 在指定父节点下创建新 GameObject.
- [ ] 支持 `insertSiblingIndex` 指定插入位置,不传时默认末尾.
- [ ] `insertSiblingIndex` 超出范围时自动修正到有效范围(末尾),与 `prefab.moveOrCopyGameObject` 行为保持一致.
- [ ] 创建名称由 `name` 参数提供,并支持设置基础属性(`tag`,`layer`,`isActive`,`isStatic`,`hideFlags`).
- [ ] 自动保存预制体并返回新对象路径与索引.

#### 命令 2: prefab.setComponentProperties

- [ ] 在指定 GameObject 的指定组件上批量修改属性.
- [ ] 支持属性路径语法与数组元素路径语法.
- [ ] 支持引用类型设置.
- [ ] 自动保存预制体并返回变更明细.

#### 命令 3: prefab.addComponent

- [ ] 给指定 GameObject 添加组件(内置组件或自定义组件).
- [ ] 支持 `componentType` 传全名或短名.
- [ ] 对不可重复组件,重复添加时返回明确错误.

#### 命令 4: prefab.deleteComponent

- [ ] 删除指定组件类型和索引的组件.
- [ ] 支持同类型多组件通过 `componentIndex` 精确删除.
- [ ] 必需组件(如 Transform)不可删除.

### 2.2 排除项

- 不支持跨预制体编辑.
- 不支持创建/删除预制体根对象.
- 不支持修改非序列化运行时字段.
- 不支持超出 Unity 序列化系统的反射黑魔法写入.

## 3. 协议总则

### 3.1 路径和定位约束

- `prefabPath` 必须是 `Assets/.../*.prefab` 形式.
- 所有 GameObject 输入定位都必须支持 `objectPath + siblingIndex` 语义:
  - `objectPath`: 必填,例如 `DialogMain/Panel_Content/ConfirmButton`.
  - `siblingIndex`: 可选,同名节点索引,从 0 开始,默认 0.
- 父节点定位使用 `parentPath + parentSiblingIndex`.
- 插入位置使用 `insertSiblingIndex`,该字段仅用于插入位置,不参与同名节点定位.

### 3.2 组件类型解析约束

`componentType` 支持两种形式:

1. 全名: `UnityEngine.UI.Button`.
2. 短名: `Button`.

解析规则:

1. 先尝试全名精确匹配.
2. 再按短名匹配.
3. 若短名匹配到多个类型,返回 `AMBIGUOUS_COMPONENT_TYPE`.

### 3.3 错误码扩展约束(完整协议优先)

本需求引入新错误码,实现时必须同步修改:

1. `Code/Assets/Editor/UnityAgentSkills/Core/AgentCommandModels.cs`.
2. `Code/Assets/Editor/UnityAgentSkills/UnityAgentSkillsPlugin.cs` 的 `IsKnownCommandErrorCode`.
3. 对应技能导出文档真源(`Code/Assets/Editor/UnityAgentSkills/SkillsExporter/SkillConfigs/SkillConfig_*.cs` 的 `SkillMarkdown`),并确认导出后的 `<exportDir>/skills/<skillId>/SKILL.md` 已同步.

新增错误码清单:

- `PREFAB_NOT_FOUND`
- `GAMEOBJECT_NOT_FOUND`
- `COMPONENT_TYPE_NOT_FOUND`
- `AMBIGUOUS_COMPONENT_TYPE`
- `COMPONENT_NOT_FOUND`
- `COMPONENT_ALREADY_EXISTS`
- `CANNOT_DELETE_REQUIRED_COMPONENT`
- `PROPERTY_NOT_FOUND`
- `INVALID_PROPERTY_PATH`
- `TYPE_MISMATCH`
- `REFERENCE_TARGET_NOT_FOUND`
- `REFERENCE_TARGET_TYPE_MISMATCH`
- `ASSET_NOT_FOUND`
- `ASSET_TYPE_MISMATCH`
- `EMPTY_PROPERTIES`

### 3.4 componentIndex 语义与 queryComponents 联动约束

- `componentIndex` 只在“同一 GameObject 上的同类型组件集合”内计数,从 0 开始.
- 计数顺序以 Unity `GetComponents(componentType)` 返回顺序为准.
- 传入 `componentIndex` 时,不会跨类型计数,例如第 2 个 `Image` 与第 2 个 `Button` 的索引独立.
- 为保证自动化可用性,调用方可先用 `prefab.queryComponents` 获取组件列表,再按同类型出现顺序推导 `componentIndex`.
- 协议升级要求: `prefab.queryComponents` 后续应补充 `componentIndex` 字段,减少调用方推导歧义.

### 3.5 旧命令错误码迁移策略

- 本文错误码清单是对新增 4 个命令的正式协议承诺.
- 存量命令若仍返回 `RUNTIME_ERROR`,在迁移完成前不视为本需求验收失败.
- 后续版本应将 `04_预制体编辑01.md` 与 `04_预制体编辑02.md` 的错误码示例逐步迁移到白名单错误码体系.

## 4. 命令参数定义

### 4.1 prefab.createGameObject

| 参数名             | 类型   | 必填 | 默认值            | 说明                                                          |
| ------------------ | ------ | ---- | ----------------- | ------------------------------------------------------------- |
| prefabPath         | string | 是   | -                 | `Assets/` 开头的预制体路径                                    |
| parentPath         | string | 否   | prefab 根对象路径 | 父节点路径                                                    |
| parentSiblingIndex | int    | 否   | 0                 | 同名父节点索引,当 parentPath 为 prefab 根对象时必须为 0       |
| name               | string | 是   | -                 | 新 GameObject 名称                                            |
| insertSiblingIndex | int    | 否   | -1                | 插入位置,-1 表示末尾                                          |
| initialProperties  | object | 否   | {}                | 可选初始属性: `tag`,`layer`,`isActive`,`isStatic`,`hideFlags` |

说明:

- 不传 `parentPath` 时,默认以 prefab 根 GameObject 作为 parent.
- prefab 根对象路径取自 `prefab.queryHierarchy` 返回的根节点 `path`(通常等于根 GameObject 的 `name`).
- 当 parent 定位到 prefab 根对象时,`parentSiblingIndex` 必须为 0.

### 4.2 prefab.setComponentProperties

| 参数名         | 类型   | 必填 | 默认值 | 说明                       |
| -------------- | ------ | ---- | ------ | -------------------------- |

| objectPath     | string | 是   | -      | 目标对象路径               |
| siblingIndex   | int    | 否   | 0      | 同名对象索引               |
| componentType  | string | 是   | -      | 组件全名或短名             |
| componentIndex | int    | 否   | 0      | 同类型组件索引             |
| properties     | object | 是   | -      | 待修改属性键值对           |

说明:

- `properties` 不能为空对象,为空时返回 `EMPTY_PROPERTIES`.

### 4.3 prefab.addComponent

| 参数名            | 类型   | 必填 | 默认值 | 说明                       |

| objectPath        | string | 是   | -      | 目标对象路径               |
| siblingIndex      | int    | 否   | 0      | 同名对象索引               |
| componentType     | string | 是   | -      | 组件全名或短名             |
| initialProperties | object | 否   | {}     | 新组件初始属性             |

### 4.4 prefab.deleteComponent

| 参数名         | 类型   | 必填 | 默认值 | 说明                       |
| -------------- | ------ | ---- | ------ | -------------------------- |

## 5. setComponentProperties 属性协议

### 5.1 属性路径语法

支持两种路径写法:

1. Unity canonical: `lootItems.Array.data[0].dropRate`.
2. 简写语法: `lootItems[0].dropRate`.

映射规则:

- 解析层先将简写转换为 canonical 后再调用 SerializedProperty.
- 当两种写法同时出现时,按 key 原文独立处理,不自动去重.

### 5.2 支持的属性类型

- 基础类型: `int`,`float`,`bool`,`string`.
- 枚举: 使用整数写入.
- Unity 常见结构: `Color`,`Vector2`,`Vector3`,`Vector4`,`Rect`,`Bounds`,`Quaternion`.
- 数组与列表: 支持整数组替换和单元素路径修改.
- 嵌套对象: 支持多级属性路径.
- 引用类型: 见 5.3.

### 5.3 引用类型统一协议

引用值必须使用 `$ref` 包装对象表示,或使用 `null` 置空.

#### A) 置空引用

```json
{
  "targetGraphic": null
}
```

#### B) 引用 prefab 内 GameObject

```json
{
  "someGameObjectRef": {
    "$ref": {
      "kind": "prefabGameObject",
      "objectPath": "DialogMain/Panel_Content/IconRoot",
      "siblingIndex": 0
    }
  }
}
```

#### C) 引用 prefab 内组件

```json
{
  "targetGraphic": {
    "$ref": {
      "kind": "prefabComponent",
      "objectPath": "DialogMain/Panel_Content/ConfirmButton/Image",
      "siblingIndex": 0,
      "componentType": "UnityEngine.UI.Image",
      "componentIndex": 0
    }
  }
}
```

#### D) 引用项目资源(Asset)

```json
{
  "sprite": {
    "$ref": {
      "kind": "asset",
      "assetPath": "Assets/.../btn_confirm.png",
      "assetType": "UnityEngine.Sprite"
    }
  }
}
```

引用校验规则:

- `prefabGameObject`: 目标对象不存在时返回 `REFERENCE_TARGET_NOT_FOUND`.
- `prefabComponent`: 对象或组件不存在返回 `REFERENCE_TARGET_NOT_FOUND`,类型不匹配返回 `REFERENCE_TARGET_TYPE_MISMATCH`.
- `asset`: 资源不存在返回 `ASSET_NOT_FOUND`,资源类型不匹配返回 `ASSET_TYPE_MISMATCH`.

## 6. 输出协议

返回结构沿用既有命令的 `results/result/saved` 框架风格,字段命名以本文定义为准.成功时 `result` 至少包含:

- `prefabPath`
- `objectPath`(或 `createdObjectPath`)
- `instanceID`
- `saved`

`prefab.createGameObject` 额外返回:

- `createdObjectPath`: 新建对象最终路径.
- `createdSiblingIndex`: 新建对象在同名节点集合中的索引,用于后续 `objectPath + siblingIndex` 精确定位.
- `insertSiblingIndexApplied`: 实际插入到父节点 children 列表中的索引.

`prefab.addComponent` 额外返回:

- `componentType`
- `componentIndex`: 新增组件在同类型组件集合中的索引.
- `componentInstanceID`: 新增组件的 instanceID.

`prefab.deleteComponent` 额外返回:

- `deletedComponentType`
- `deletedComponentIndex`
- `deletedComponentInstanceID`

`prefab.setComponentProperties` 额外返回:

- `componentType`
- `componentIndex`
- `componentInstanceID`: 被修改组件的 instanceID.
- `modifiedProperties`: 数组,每项包含 `name`,`oldValue`,`newValue`.

说明:

- `instanceID` 默认指 `objectPath` 对应 GameObject 的 instanceID.
- 当结果中返回 `createdObjectPath` 时,`instanceID` 指 `createdObjectPath` 对应 GameObject 的 instanceID.
- 组件级 instanceID 使用 `componentInstanceID`/`deletedComponentInstanceID` 承载.
- 引用类型在 `modifiedProperties.oldValue/newValue` 中的序列化形态以通用序列化输出为准,不承诺回显 `$ref`.

## 7. 示例

### 7.1 创建对象

```json
{
  "id": "cmd_create",
  "type": "prefab.createGameObject",
  "params": {
    "prefabPath": "Assets/.../DialogMain.prefab",
    "parentPath": "DialogMain/Panel_Content",
    "parentSiblingIndex": 0,
    "name": "NewButtonRoot",
    "insertSiblingIndex": 0,
    "initialProperties": {
      "tag": "Untagged",
      "layer": 5,
      "isActive": true
    }
  }
}
```

### 7.2 添加组件并初始化

```json
{
  "id": "cmd_add_button",
  "type": "prefab.addComponent",
  "params": {
    "prefabPath": "Assets/.../DialogMain.prefab",
    "objectPath": "DialogMain/Panel_Content/NewButtonRoot",
    "siblingIndex": 0,
    "componentType": "UnityEngine.UI.Button",
    "initialProperties": {
      "interactable": true
    }
  }
}
```

### 7.3 修改组件属性(含数组和引用)

```json
{
  "id": "cmd_set_props",
  "type": "prefab.setComponentProperties",
  "params": {
    "prefabPath": "Assets/.../DialogMain.prefab",
    "objectPath": "DialogMain/Panel_Content/NewButtonRoot",
    "siblingIndex": 0,
    "componentType": "UnityEngine.UI.Button",
    "componentIndex": 0,
    "properties": {
      "interactable": false,
      "colors.normalColor": { "r": 1, "g": 0, "b": 0, "a": 1 },
      "targetGraphic": {
        "$ref": {
          "kind": "prefabComponent",
          "objectPath": "DialogMain/Panel_Content/NewButtonRoot/Image",
          "siblingIndex": 0,
          "componentType": "UnityEngine.UI.Image",
          "componentIndex": 0
        }
      }
    }
  }
}
```

### 7.4 删除组件

```json
{
  "id": "cmd_del_component",
  "type": "prefab.deleteComponent",
  "params": {
    "prefabPath": "Assets/.../DialogMain.prefab",
    "objectPath": "DialogMain/Panel_Content/NewButtonRoot",
    "siblingIndex": 0,
    "componentType": "UnityEngine.UI.Button",
    "componentIndex": 0
  }
}
```

## 8. 验收清单

### 8.1 协议一致性

- [ ] `prefabPath` 非 `Assets/` 开头时报 `INVALID_FIELDS`.
- [ ] `prefabPath` 非 `Assets/.../*.prefab` 形态时报 `INVALID_FIELDS`.
- [ ] 所有需要定位对象的命令都支持 `objectPath + siblingIndex`.
- [ ] create 命令使用 `insertSiblingIndex`,不复用 `siblingIndex`.

### 8.2 错误码扩展

- [ ] `AgentCommandModels.cs` 增加本需求新增错误码.
- [ ] `IsKnownCommandErrorCode` 白名单同步新增错误码.
- [ ] 抛出 `CODE: message` 可在结果中得到同名 `error.code`.
- [ ] 新增 4 个命令必须使用扩展错误码,不得降级为笼统 `RUNTIME_ERROR`.
- [ ] 存量命令按迁移策略逐步对齐,迁移完成前不阻断本需求验收.

### 8.3 功能正确性

- [ ] create/add/set/delete 四类命令均可独立执行成功.
- [ ] 同名对象与同类型多组件场景下可精确定位.
- [ ] 数组和嵌套属性修改正确.
- [ ] `prefabGameObject`,`prefabComponent`,`asset` 三类引用写入正确.
- [ ] 删除 Transform 返回 `CANNOT_DELETE_REQUIRED_COMPONENT`.

### 8.4 回归验证

- [ ] 与 `prefab.queryHierarchy` 联动可验证对象结构变化.
- [ ] 与 `prefab.queryComponents` 联动可验证组件和属性变化.
- [ ] 现有命令 `prefab.setGameObjectProperties`,`prefab.deleteGameObject`,`prefab.moveOrCopyGameObject` 行为不被破坏.

## 9. 实现注意事项

- 建议统一采用 `LoadPrefabContents -> 修改 -> SaveAsPrefabAsset -> UnloadPrefabContents` 模式.
- `prefab.setComponentProperties` 需明确路径解析与类型转换日志,便于定位失败点.
- 组件类型解析发生短名歧义时,必须显式报错,禁止静默取第一个.
- 文档中所有错误码均为正式协议承诺,实现不得降级为笼统 `RUNTIME_ERROR`.
